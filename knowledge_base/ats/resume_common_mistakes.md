# Common Resume Mistakes

These issues commonly reduce recruiter interest or ATS compatibility.

---

# Generic Resume

Problem:
Using the same resume for every application.

Why it matters:
Recruiters expect resumes to match the job description.

Recommendation:
Tailor your skills, projects, and summary for each application.

---

# Weak Professional Summary

Problem:
Writing vague statements such as:

"Hardworking individual looking for opportunities."

Recommendation:
Clearly summarize:

- Experience level
- Technical domain
- Key skills
- Career objective

---

# Listing Responsibilities Instead of Achievements

Weak:

"Responsible for developing web applications."

Strong:

"Developed a Flask-based web application used by over 300 users."

Achievements demonstrate impact.

---

# Missing Quantifiable Results

Whenever possible include measurable outcomes.

Examples:

✓ Improved inference speed by 30%

✓ Reduced API latency by 45%

✓ Increased test coverage from 60% to 92%

Numbers strengthen credibility.

---

# Poor Project Descriptions

Weak:

"Made a chatbot."

Better:

"Built a Retrieval-Augmented Generation chatbot using Gemini, ChromaDB, and Streamlit capable of answering document-based questions."

Include:

- Technologies
- Problem solved
- Personal contribution
- Results

---

# Keyword Stuffing

Adding dozens of unrelated keywords reduces readability.

Bad:

Python Java SQL AWS Azure Kubernetes Docker Machine Learning AI React Flutter Android iOS ...

Instead:

Naturally integrate relevant skills into projects and experience.

---

# Inconsistent Formatting

Examples:

- Mixed bullet styles
- Different fonts
- Uneven spacing
- Different date formats

Consistency improves readability.

---

# Missing Technical Links

Technical resumes should include:

- GitHub
- LinkedIn
- Portfolio
- Personal website (optional)

Broken links should be removed.

---

# Weak Skills Section

Avoid listing every technology ever used.

Instead:

Group skills.

Example:

Programming:
Python, Java, C++

Frameworks:
Flask, Streamlit

Databases:
MySQL, PostgreSQL

AI/ML:
Scikit-learn, TensorFlow

---

# Overly Long Resume

Recruiters spend limited time reviewing resumes.

Keep content concise.

Every bullet should communicate value.

---

# Grammar and Spelling Errors

Frequent grammar mistakes reduce professionalism.

Always proofread before submitting.

---

# Missing Action Verbs

Weak:

"Worked on website."

Better:

"Developed a responsive portfolio website using React."

Start bullets with strong action verbs.

---

# Lack of Focus

Do not mix unrelated career goals.

Example:

Avoid combining:

- Web Development
- Robotics
- Finance
- Graphic Design

Unless genuinely applying across multiple domains.

A focused resume appears stronger.

---

# Common Resume Red Flags

- No measurable achievements
- Empty project descriptions
- Unprofessional email address
- Inconsistent formatting
- Missing GitHub for software roles
- Missing LinkedIn
- Walls of text
- Excessive buzzwords
- Irrelevant personal information
